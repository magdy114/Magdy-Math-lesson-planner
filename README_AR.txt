Magdy Lesson Planner - AI School Edition
========================================

هذه النسخة مصممة لتعمل كرابط واحد لكل المعلمين.
المعلم لا يرى API Key ولا يحتاج إلى وضعه، ولا يوجد كود دخول.

الفكرة الصحيحة:
1) تضع التطبيق على سيرفر واحد فقط.
2) تضع OPENAI_API_KEY داخل ملف .env على السيرفر.
3) ترسل رابط التطبيق للمعلمين.
4) كل المعلمين يستخدمون نفس المفتاح الموجود على السيرفر بدون أن يظهر لهم.

مهم جدًا:
لا تضع ملف .env الحقيقي داخل النسخة التي ترسلها للمعلمين.
لا تكتب API Key داخل app.py.
إذا أرسلت المفتاح داخل ZIP يستطيع أي شخص استخراجه.

تشغيل محلي للتجربة:
-------------------
1) فك الضغط عن الملف.
2) انسخ .env.example واجعل اسمه .env
3) ضع API Key داخل .env:
   OPENAI_API_KEY=your_key_here
4) اضغط START_APP.bat
5) افتح http://127.0.0.1:5000

تشغيل كسيرفر للمعلمين:
----------------------
ضع هذه الملفات على سيرفر Python/Flask مثل Render / Railway / Replit / VPS.
ضع المتغيرات في إعدادات السيرفر:
OPENAI_API_KEY
OPENAI_MODEL
DAILY_TOTAL_LIMIT
DAILY_USER_LIMIT

الموديل:
--------
الموديل الافتراضي في .env.example هو gpt-5.5.
يمكنك تغييره من OPENAI_MODEL دون تعديل الكود.

الحماية:
--------
DAILY_TOTAL_LIMIT: حد يومي عام لكل التطبيق.
DAILY_USER_LIMIT: حد يومي لكل معلم حسب اسمه/IP.

المخرجات:
---------
- ملف Word DOCX بنفس قالب المدرسة الرسمي.
- عند إضافة أكثر من درس، يتم تنزيل ZIP.
- كل الملفات تحفظ تلقائيًا في generated_lessons.
- الأخطاء تحفظ في logs/error_log.txt ولا يغلق التطبيق عند حدوث خطأ.

تعديل هذه النسخة:
----------------
- تم حذف خانة كود الدخول نهائيًا.
- تاريخ الحصة أصبح اختياريًا ولا يمنع التوليد.
- الشريط العلوي وبيانات الخطة مثبتة أثناء التمرير.
- بقي اسم المعلم مطلوبًا حتى يظهر صحيحًا في القالب الرسمي.

المتطلبات:
----------
Python 3.10 أو أحدث.
